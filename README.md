# Contemporary-Project-Management-Agile-Scrum-Methodology
Contemporary Project Management Agile Scrum Methodology using JIRA Platform
## My Role – Developer

In this project, I worked as a **Developer** in an Agile Scrum team.

The **Product Owner, Jasmine**, gathered the client's requirements, created user stories, and prioritized the product backlog. The **Scrum Master, Ankit**, ensured that the Scrum process was followed by conducting sprint planning, daily stand-up meetings, sprint reviews, and retrospectives.

As a developer, my main responsibility was to implement the user stories assigned to me during each sprint. I updated my tasks in Jira, collaborated with my teammates, and ensured that the assigned features were completed within the sprint timeline.

At the end of every sprint, we participated in a **Sprint Review** with the client to demonstrate the completed user stories and collect feedback. Any new requirements or changes requested by the client were added to the product backlog and prioritized for future sprints. If a user story was not completed within a sprint, it was carried forward and completed in the following sprint.

During one of the sprints, I was responsible for identifying, fixing, and testing a reported bug before updating its status in Jira.

Working on this project gave me practical experience with the **Agile Scrum framework**, Jira project management, sprint-based development, teamwork, and delivering software based on continuous client feedback.
